Reverse Polish Notation – Infix to Postfix Notation

2010-12-27
  • Algorithms
    • Basic Theory

  • Theory

本文仅介绍中缀表达式转换为后缀表达式的原理。

例子： A+B(C-D)-E/F => ABCD-+EF/-

在中缀表达式转换为后缀表达式的过程中，操作数的顺序是一致的。转换的过程首先自左向右读取中缀表达式，遇到操作数就直接输出。遇到操作符时，把操作符压入符号栈，根据优先级比较后决定是否弹栈。

  • isp: in stack priority. 栈内优先级
  • icp: in coming priority. 栈外优先级

    优先级表格：

操作符 # ( ^ * / % + - )
isp 0 1 7 5 3 8
icp 0 8 6 4 2 1

Basic Theory: Linux System

Chapter 1

1. GNOME: GNU Network Object Model Environment; KDE: K Desktop Environment, which is based on Nokia QT library.

2. Nautilus is the default File Manager in GNOME, the same as Explorer in Microsoft Windows.

3. IDE hard disk is named as ‘hdx’, SCSI hard disk is named as ‘sdx’.

 

Chapter 2

普通PC有两个IDE接口，最多可挂载四块硬盘，分别被识别为hda、hdb、hdc、hdd；由于PC的设计，每块硬盘最多可分四个主分区，分别为hdx1、hdx2、hdx3、hdx4；可以将其中一个主分区设置为扩展分区，在扩展分区内建立若干逻辑分区；由于系统最多共有四个主分区，故逻辑分区的编号从5开始，即hdx5。

Linux下的swap分区是所有分区中唯一一个不需要mount的分区；mount的目的是将分区挂载到目录，由于在Linux里没有盘符的概念，所以通过访问目的达到访问分区内容的目的。

推荐的分区划分方案：

分区格式                   挂载点               建议大小

ext3(Linux native)     /boot      100MB左右            
swap                                        物理内存两倍

ext3(Linux native)     /              剩余空间

其他常用目录： /usr   /home   /root   /var   /tmp

 

Chapter 3

X Window 采用 Client/Server 模式，具有平台无关、网络透明等特性；其中 X Server 跟踪输入设备，并提供显示功能；X Client 使用 X Server 所提供的功能显示运行结果。

1. X  协议（负责 X Client 与 X Server 之间的通信，可用于实现远程桌面）
2. Xlib 函数接口（X Window 系统程序界面，应用程序通过调用该层函数实现界面）
3. Window Manager 窗口管理器

 

Chapter 4

系统的三个标准文件：标准输入(STDIN(0))、标准输出(STDOUT(1))、标准错误输入(STDERR(2))。

Linux 重定向可以将文件标准输出重定向到文件，也可以将一个文件作为另一程序的输入。
>符号：将标准输出重定向到某一文件，并覆盖原内容   如： ls -l > list.info
>>符号：重定向标准输出时将内容接在目标文件后       如： ls -l >> list.info
<符号：将某一文件作为重定向后程序的标准输入         如： grep keyword < info.txt
|管道符：将左边的输出作为右边命名的输入                如： ls -a | grep /home
过滤器：接受标准输入，转化后写入标准输出              如： ls- a | tee grep /home

About Data Structure

2010-12-25
  • Basic Theory
    • Programming
  • Data Structure
  • Theory

是数据而不是算法是系统构造的关键因素。

数据结构是计算机存储、组织数据的方式。通常情况下，精心选择的数据结构可以带来更高的运行或者存储效率的算法。数据结构往往同高效的检索算法和索引技术有关。

Sartaj Sahni 在他的《数据结构、算法与应用》一书中称：“数据结构是数据对象，以及存在于该对象的实例和组成实例的数据元素之间的各种联系。这些联系可以通过定义相关的函数来给出。”他将数据对象（data object）定义为“一个数据对象是实例或值的集合”。
Clifford A.Shaffer 在《数据结构与算法分析》一书中的定义是：“数据结构是 ADT（抽象数据类型 Abstract Data Type） 的物理实现。”
Lobert L.Kruse 在《数据结构与程序设计》一书中，将一个数据结构的设计过程分成抽象层、数据结构层和实现层。其中，抽象层是指抽象数据类型层，它讨论数据的逻辑结构及其运算，数据结构层和实现层讨论一个数据结构的表示和在计算机内的存储细节以及运算的实现。

一般认为，一个数据结构是由数据元素依据某种逻辑联系组织起来的。对数据元素间逻辑关系的描述称为数据的逻辑结构；数据必须在计算机内存储，数据的存储结构是数据结构的实现形式，是其在计算机内的表示；此外讨论一个数据结构必须同时讨论在该类数据上执行的运算才有意义。

在许多类型的程序的设计中，数据结构的选择是一个基本的设计考虑因素。许多大型系统的构造经验表明，系统实现的困难程度和系统构造的质量都严重的依赖于是否选择了最优的数据结构。许多时候，确定了数据结构后，算法就容易得到了。有些时候事情也会反过来，我们根据特定算法来选择数据结构与之适应。不论哪种情况，选择合适的数据结构都是非常重要的。

选择了数据结构，算法也随之确定，是数据而不是算法是系统构造的关键因素。这种洞见导致了许多种软件设计方法和程序设计语言的出现，面向对象的程序设计语言就是其中之一。
