Basic Theory: Linux System

2010-12-25
  • GNU/Linux
    • Programming
  • GNOME
  • KDE
  • Linux
  • Theory
  • X Window

Chapter 1

1. GNOME: GNU Network Object Model Environment; KDE: K Desktop Environment, which is based on Nokia QT library.

2. Nautilus is the default File Manager in GNOME, the same as Explorer in Microsoft Windows.

3. IDE hard disk is named as ‘hdx’, SCSI hard disk is named as ‘sdx’.

 

Chapter 2

普通PC有两个IDE接口，最多可挂载四块硬盘，分别被识别为hda、hdb、hdc、hdd；由于PC的设计，每块硬盘最多可分四个主分区，分别为hdx1、hdx2、hdx3、hdx4；可以将其中一个主分区设置为扩展分区，在扩展分区内建立若干逻辑分区；由于系统最多共有四个主分区，故逻辑分区的编号从5开始，即hdx5。

Linux下的swap分区是所有分区中唯一一个不需要mount的分区；mount的目的是将分区挂载到目录，由于在Linux里没有盘符的概念，所以通过访问目的达到访问分区内容的目的。

推荐的分区划分方案：

分区格式                   挂载点               建议大小

ext3(Linux native)     /boot      100MB左右            
swap                                        物理内存两倍

ext3(Linux native)     /              剩余空间

其他常用目录： /usr   /home   /root   /var   /tmp

 

Chapter 3

X Window 采用 Client/Server 模式，具有平台无关、网络透明等特性；其中 X Server 跟踪输入设备，并提供显示功能；X Client 使用 X Server 所提供的功能显示运行结果。

1. X  协议（负责 X Client 与 X Server 之间的通信，可用于实现远程桌面）
2. Xlib 函数接口（X Window 系统程序界面，应用程序通过调用该层函数实现界面）
3. Window Manager 窗口管理器

 

Chapter 4

系统的三个标准文件：标准输入(STDIN(0))、标准输出(STDOUT(1))、标准错误输入(STDERR(2))。

Linux 重定向可以将文件标准输出重定向到文件，也可以将一个文件作为另一程序的输入。
>符号：将标准输出重定向到某一文件，并覆盖原内容   如： ls -l > list.info
>>符号：重定向标准输出时将内容接在目标文件后       如： ls -l >> list.info
<符号：将某一文件作为重定向后程序的标准输入         如： grep keyword < info.txt
|管道符：将左边的输出作为右边命名的输入                如： ls -a | grep /home
过滤器：接受标准输入，转化后写入标准输出              如： ls- a | tee grep /home

#filters: expand, sed, awk, fmt, tac, tr, grep, nl, pr, tee
