What’s happened in the last 2 months?

2011-05-30
  • Kevin's Life
    • Rework
      • Summary
  • Business
  • Interview
  • Rework
  • Wedding

还记得上一次更新是愚人节那天，这两个月一转眼就过去了。这段时间没有什么重要的更新，也没有另外赚到钱；但仍然有一些事情、一些思考，值得记录下来。

在写这篇日志的时候，我努力却始终回想不起来四月份做了哪些事情，只好借着QQ的聊天记录和邮件的历史，才隐隐约约有了个大概。

四月份里啸哥办了喜事，成为我们一班里面最先结婚的成功人士。结婚当天来了好多大学同学，大家吃吃喝喝、玩玩乐乐、其乐融融。有意思的是婚礼上放的一个祝福短片，里面有汤包（澳大利亚）、老陈（美国）、老曾（日本），好像是特意从每个国家里挑一个，把正在国外读书的同学都拍了进来，派头十足。大家还谈到后面啸哥要去马尔代夫度蜜月，五天就得花掉四五万，也算是烧钱烧到令人嫉妒的事情了。当然我自己的生活还是在平淡中继续着，烧钱的唯一条件就是得有钱，这点我从来不奢望。

老胡在准老丈人的资助下贷款买房子了，所以他和他美眉结婚也就是时间问题而已。陈梁因为换工作，五月份搬到金桥这边了；周末吃饭什么的，可以多点几个菜然后AA，很不错。四月份自己算是比较混日子的过着，在vWorker上面做了一个小项目，然后又Cancel了一个大项目，基本算是没赚着钱了。

四月份也是IT公司涨薪和招人的月份，我接到百度上海的电话，去面试了一趟，还算有点感触。百度面试的时候问了一个分制的算法问题，然后就是一些内存、调度等细节的技术问题；可以说百度对工程师的基础技术要求还是比较高的。因为我也没准备去那边工作，所以表明了态度是去体验的，再说百度公司被骂的太厉害了，诸多原因，敬而远之。后面还收到了微软和谷歌的邮件，招聘的职位都是我完全不擅长的，再加上我对自己技术能力的了解，所以就没有再考虑了。

四月底的时候在vWorker上面遇到了一个叫LDApp的公司，随后是两个几万美金的大项目，我看着眼馋就跟进了。当然大项目对bid的要求也很高，一直让我写proposal，就这样，我五月份都在写这两个项目的文档。从这件事情上，我发现自己还是挺爱财的，就是有时候比较迷糊。我到现在也没搞清楚对方是不是骗子，我能确定的是自己写设计文档的能力肯定是进步了；吃一堑长一智吧，本来赚钱也没那么容易，如果真是骗子，我是已经有了心理准备的。

五月份已经将近结尾，虽然没有赚着钱，但是还是很充实的度过了。话虽如此，目前需要的正是多赚点钱入手T410s，旧的笔记本实在太差劲了，我几乎没办法在家里办公。至于怎么赚钱，途径很多，当然最简单的还是多做vWorker的项目，另外自己也在做whisperlove的网站，总之希望自己在八月份的时候可以入手新笔记本！

从目前的尝试来看，最容易赚的钱都是些零花钱（一千块左右），而且是几乎没有资产回报的，这当然不是好的模式。在我看来，一个好的模式必须有两个特点：巨大的潜在盈利空间，即使项目是一次性的；持续的资产回报，即使盈利能力有限。有一个认知上的收获，可以总结为：小公司也可以做大项目。我想无论是做生意还是做项目管理，都需要积累经验，积累到了一定程度，做事终会变得熟练。

Taking a break and thinking about next step

2011-04-01

韩寒博客里的歌真不错，就这样开着网页，只为听歌，然后一边写博客。

在愚人节这天写博客本觉得有点怪，我并不是故意的挑这个时间，仅仅是时间刚好安排在这天而已。但又想生活的故事其实跟愚人节里的桥段一样，起起落落、充满戏剧性，这样说来，我选择在今天来记录最近的生活，也就合情合理了。

最近并没有什么特别难过的事情，虽然大多事情也并不是一帆风顺，有些磕磕绊绊，但结局总归还算圆满，对于这样我已经很满意了，这些不愉快的过程，我也并不完全认为是坏事。

要说写这篇博客的最大动力，我觉得可以用“该‘换档’”来概括。我形容自己就像开手动档的汽车，如果把油门踩到底了，就需要及时的换档，否则就会出故障。人的身体跟汽车一样，忙了一段时间就要换档调整，虽然暂时会变得迟缓，可这是达到更高速度的必须经过。这篇博客是用来对最近的工作做一些归纳，我也顺便调整下自己的状态，为后面的发展做准备。

三月份里，公司的项目基本算是清闲了，我接连为Kodak Share Button App v3.0/v3.1做了两个新站点（Target.com 和 Walmart.com），前一个由于Kodak Gallery RESTful API不完善而推迟，后一个因为涉及到与HP的法律纠纷问题也延期了。虽然这个月为项目写了很多功能代码，后来连BUG都改完了，但是对公司而言，还没有产生任何的商业价值。对于这一点，我只能说是我的运气太差。

在vWorker上的项目，三月份的大多也都完成了，收入量相对一月份也有较多的提升。其中最废时的一个项目是和一位旅美的日本人合作，由于没有在bid前检查他的rating，轻易的接下了项目，后面在合作和沟通上也有一些差错，工作量很小的项目拖了差不多一个月的时间，这或许是我这个月在vWorker上最大的教训。而在后面，我主要集中在做WP7手机相关的项目，这也正符合我的想法，更加专注移动应用领域。

我认为在项目合作上，第一印象对双方都非常重要，这往往是长期合作的重要基础。除了手机应用意外，一些跟本地化、产品推广的任务我也有涉及，相对于编码的工作，这些工作的盈利更快、利润空间更大，因而客户资源也更加宝贵。

接下来的一段时间里，可以学学 iphone 和 android 开发，有了借来的iphone，要好好利用，另外，晚上抽空去跑跑步，注意饮食卫生，最近不能吃的东西太多了，健康的身体是奋斗的本钱，调节好身体，为后面做准备。

This paragraph is about starting up a website by buying and setting up a VPS.

We bought a ‘.com’ domain at http://www.net.cn with 139CNY/year, and a VPS from Burst.NET with 59.5USD/year. One tip is that Burst.NET supports user to choose the geographical position of VPS, 2 options are provided: SCRANTON and Los Angeles, as we tested, SCRANTON is a little faster than LA with either Shanghai or Rochester network. So if you want to buy your own VPS from Burst.NET, we suggest you to choose SCRANTON. But if you want to test speed by yourself, here is a bit more info to you:

Ping/Traceroute Test:
66.96.192.92 (Scranton)
184.82.225.2 (California)
178.238.129.83 (EU/UK)

Download Test:
http://66.96.192.92/bigtest.tgz – Scranton, PA USA
http://184.82.225.2/bigtest.tgz – Los Angeles, CA USA
http://178.238.129.83/bigtest.tgz – Manchester, UK

You may chose a Ubuntu 10.4 LTS 32bit system, the VPS had set up a LAMP environment so that you can easily install your web system. The web root is under ‘/var/www’ by default, if you place a php script there, it will run up when  visited via web browsers.

To access your VPS with SSH, we suggest you to use Putty which is a popular and lightweight tool with multi platform support; To copy files from local to your VPS, you may need WinSCP via SSH or Filezilla via FTP, and you need set port to 22 for both before use; To remote access MySQL database, we suggest Navicat which has powerful features and friendly user interface.

By default, VPS don’t have GCC or wget installed, you may setup them very easily with ‘apt-get install’; if you want download a package to your host, you don’t need to get it back to local and then copy to the remote, you may just use ‘wget –c http://www.sample.com/xxx.tar.gz’, the download speed of VPS network should be much faster than local.

Finally, if you got any problem when set up your VPS, you can use helpdesk by Burst.NET, before place your issues, you may need a registration first.

 

This paragraph is about configuring mail system on your VPS, my solution is postfix + dovecot + afterlogic.

I want to notice you that my experience is only valuable for Ubuntu 10.4, this tutorial may not work with other versions of Ubuntu!

Basically, I setup postfix and dovecot by following the official guidelines(I II), you may following them first, and then I will figure out some common issues with my work around.

1. enable log for postfix, postfix uses system log, and rely on rsyslog service, so if you can’t find mail log files ‘/var/log/mail.*’, you may check installation of rsyslog and ensure it’s activated. See here for more info on debugging postfix.

2. add a mail user, the mail service use system user mechanism, you may add a new mail account with ‘useradd xxx –g mail –s /usr/sbin/nologin’, and set password via ‘passwd xxx’; if system don’t automatically create ‘/home/xxx’, you may create it yourself, and set proper permission with ‘chmod’ and ‘chown’.

3.a bug on configure dovecot profile, if you error log shows ‘Plugin cmusieve not found from directory /usr/lib/dovecot/modules/lda’, you may reconfigured dovecot profile, and install sieve on your system, see this article.
To me, the work around don’t fix my problem, I did the same configuration on ‘/etc/dovecot/conf.d/01-dovecot-postfix.conf’ too.

Here is a list of sample configuration files by myself:
/etc/postfix/main.cf
/etc/dovecot/dovecot.conf
/etc/dovecot/conf.d/01-dovecot-postfix.conf

You can setup a MUA such as Outlook for test, in server side you may trace the error log to gather more error info, you may also use ‘telnet’ to test SMTP and POP3 service separately.

The last thing is about setting up web mail system, I chose AfterLogic web mail system. You can complete most deployment by following the instruction) at official website; the only issue is about mapping email address format to system account format, since my web mail attempted to login with a mail address, such as ‘xxx@reworkstudio.com’, but in fact I just need ‘xxx’ part.

The AfterLogic helpdesk figured out my mistake, I changed the settings.xml) and set UseLoginAsEmailAddress to ‘Off’, then my web mail worked.

If you have any questions, you may place a topic at AfterLogic helpdesk system, guys there are very kindly to help you out the problem.

 

That’s all, thanks for your reading.

Happiness is not easy – 2011 yearly planning

2011-02-10
  • Kevin's Life
    • Plan
  • 2011 Plan
  • Happiness

幸福没有那么容易，她就像玻璃瓶一样，美丽却又易碎。2011年的起点，我要为幸福堆砌实而不华的文字，为幸福积攒每一点点的努力，不抛弃，不放弃！

想象中的幸福是个短暂的东西，她美得惊艳，却非常短暂；而生活中的幸福是个成长的过程，她来自于每一个细节，简单真实，触手可及。

我从来不批判自我思考，即使我会重复再三的思考同一个问题；对于新的一年，我决心抛弃那些过于理想化的东西，因为他们不切实际、缺乏基础，我决心活的更现实一点，虽然简单，至少每一点的努力，都可以看得见、摸得着。

生活在理想中的人偶尔会由衷的感到满足，生活在现实中的人会实实在在的触摸到幸福，再美好的虚幻也会结束，我应该责备自己，一手制造了理想与现实的强烈对比，生活在幻想和现实的频繁切换之间，烦乱，惊慌。

现实的幸福需要很多东西：善待生命、珍惜时间、健康的身体、健全的情感、经济基础、发展追求，可能还有很多我没有想到的方面，以后慢慢补上。

生命

在我看来，我们的社会并不懂得善待生命，可能是因为人口数量大的缘故，人的生命变得廉价，对于因食品问题、拆迁问题、腐败问题、施工问题导致人死亡的事故，早已成为主流媒体每天新闻内容的常客，这种长时间的媒体关注而社会不关注的现象，反而让观众产生了麻木、漠视的错误心态。这一点上，相信我的观点和很多朋友类似：我们会更加关注自身，关注周边，我们确信自己没有能力去批判社会、帮助受害群众，但我们愿意珍视每一个身边的人、动物，善待生命。

时间

人的一辈子很短，因为人们大都不会仔细的生活，年轻时候没有感觉到时间的流逝，时间稍纵即逝，一辈子很快就到了尽头。有时候我会想，倘若我的生命即将停止，我定会珍惜与家人在一起的每一分钟，那时的我，肯定会遗憾当初，没有多一点点的努力奋斗，早一天空出时间陪伴家人。类似的文字并不新鲜，看过《钢铁是怎样炼成的》也都会有过大致相同的思考。

健康

这或许是我最头疼的话题，有句俗话说“三十岁以前你找病，三十岁以后病找你”，而且我也十分同意“身体是革命的本钱”这句话，我想现在的身体应该是亚健康到不健康之间的某个地方，如果不注意锻炼、饮食、起居，不久应该会病来如山倒吧。2011年初，我准备在健身方面投资一些，哑铃、仰卧起坐椅、运动装（热身服），利用春夏两季的业余时间锻炼身体；还有需要主要生活细节，防寒、衣被、房间的整洁等等；另外是饮食方面，周末可以自己做汤，补充营养。最主要的是，不做伤害身体的事情，珍视身体犹如珍视生命。

情感

情感的维持是2010年最糟糕的事情，相信2011肯定会变好。我需要主动和曾经熟知的好友联系，恢复正常的交际，也需要主动认识新的朋友，多扩充自己的人脉，这也就变相的扩充了我的认知领域；我需要把爱情的准则从浪漫、完美的幌子里面挣脱出来，停止对于爱情美妙的幻想，找一个可以结婚生孩子、过柴米油盐日子的女人；亲情是牢固的，必要的通讯可以消除父母的担忧，我眼里的亲情是一条橡皮筋，远点近点都可以扯在一起，它存在，而且很自然、很轻松。

经济

或者更确切的说，应该是理财。仅仅靠工资收入的是死水，忙碌一个月是那么多，闲着一个月也是那么多，与其等待公司每年那一两次的提薪机会，何不自己给自己发工资呢。很多朋友都有创业的想法，但大都缺乏基础，我更愿意做一个折中的人，在保持一份工作的同时，自己通过各种渠道赚点外快，如果一个月能多出一千块钱，大概就算提前拿到了一两年后的工资了。积水成海、积石成山，慢慢的提高自己的收入，但并不意味我要安守本分，如果有机会能做出有市场的软件，那是长期收益的项目，是更好的选择。对于基金、股票的投资是传统意义上的理财手段，今年我并不会花太多精力在学习这些理财工具上；在我看来，工具、能力提高方面的投资，更加重要，年中我会入手一个高配的笔记本用于做外快，入手一个新手机用于手机软件开发，年末的时候也希望抽出几千块钱，给家里更换点电器，等离子电视、智能洗衣机什么的。至于去新东方学口语，那是下半年的内容。另外，年中放假会回家办理护照和港澳通行证，目前Paypal依然是我的小存钱罐，有了港澳通行证就可以方便的提现了。

发展

很多中国人不知道自己的理想是什么，自然也就不知道规划自己的人生，或者简单的把赚更多钱、做更大的官、找更漂亮的老婆作为自己的发展追求，当然，我没有资格去批判这一切，在理想面前，我们大多数年轻的八年后都显得卑微不堪。一群人用一家三代人的钱去买房，更用一辈子的时间去还房贷，一辈子住在银行买的房子里，每个月都要上交类似保护费的房贷。观察周围平庸生活着的同事、朋友，仿佛看到了自己未来的缩影，他们给我一种很强烈的感觉：中国人的幸福是苦中作乐，是在不如意者十之八九中挤出来。大多人会告诉我，这样的生活并不糟糕，这就是真实的生活；我想我会接受这个残酷的现实，但也不至于完全现实到平庸的过一辈子，我可以踏实的往一个方向走，或者更确切的说，我期待自己写出有用户群的软件。单从本年的计划来说，我没有读在职研究生的计划，也没有考驾照的计划，这些事件按顺序排着，如果一直依次在做，那我也就一步步的走向平庸了，不过还好，至少是现实的生活着。

梦想要经常被拾起，因为你要记得自己还有梦想，即使你已经是一个“老男孩”。

第一次关注《老男孩》是在前不久《波士堂》的那期节目上，节目一开始，优酷的总裁古永锵跟筷子兄弟一起合唱了《老男孩》的主题曲。我很喜欢这首歌的旋律，我也因此去优酷上看了同名的电影。片子不长，差不多45分钟，但在最后情节和旋律的双重炮轰下，我还是泪崩了。

整个故事在观众心里成为了各自青春的载体，不同的人看了会有不同的故事的联想，但应该都会是与青春、理想有关的。其实此类的电影不在少数，《老男孩》的不同在于它的形式，首先它是我看的第一部在优酷投资、首映的电影，电影成本小但是故事完整，内容平淡但情节起伏，特别是最后的电视秀那一段，相信让大多以为故事会草草收场的观众喜出望外，却又为音乐感染而泪奔，故事也从简单的搞笑升华为对青春、理想的追悼、缅怀。

其实豆瓣上也有很多负面评论，可以归纳为情节老套、立意龌龊两类。我觉得情节方面不可苛求，本来就是小成本电影，加上片尾的那段煽情表演，完全可以让观众忘记前部分故事表述的平淡，相反，那些简单、通俗的情节反而成为了情节转变的铺陈。立意方面，不可否认故事再次使用了感情绑架，特别是在中国这种青春注定蹉跎的成长环境下，绑架青春理想稍显龌龊；看这部短片，可以让大部分普通的80后找到自己的影子，恋爱挫折的故事情节实际上只是一个引子，影射的是青春时期的理想；但事情总可以分两面去分析，电影成功的勾起了奔三青年对理想的追忆，让观众反思自己走过的路，拾起曾经的美好憧憬；一路走来，大部分的人在体制的影响下慢慢变得普通，对吾等22/23左右的年轻人，经常有此类影片催泪提醒，未必是一件坏事。

我一直保持一个中立的观点去看待问题，其实豆瓣上的评论十分精彩，从一颗星到五颗星的，比比皆是。观点中立并不代表自己是盲从，在此写几句话提醒自己：[1]不一定要参考别人的做法，参考意味着思维取向，意味着扼杀自己的想法和创新；[2]大多数人的做法不一定是对的，因为大多数人永远都是最平庸的；[3]梦想应该珍藏，因为梦想往往很难实现，但梦想也要经常被拾起，因为你要记得自己还有梦想，即使你已经是一个“老男孩”。

Author: Doug Fleischman, Eastman Kodak Company – 04/18/07

In the Managed World, a free tool called Ncover (from www.ncover.org) allows us to perform coverage analysis in a simple inexpensive manner. Please be aware that there is another tool also called Ncover (from sourceforge) that works in a very different way.

What is nice about Ncover is that you do not need to instrument the code, and you can run it on Release executables as long as you have the companion PDB files.

After installation, add the directory location where Ncover is installed to your System Path environment variable. Running Ncover is simple, just run it at the command line with Nunit as the parameter. (Note: If Nunit is not in your system path value, you will need to use the full path) For example, for Pelican I can do this:

When I hit return I will get the following messages:

![clip_image002[5]](http://kevinxi.files.wordpress.com/2010/12/clip_image00251.jpg)

Now Nunit will come up:

![clip_image002[7]](http://kevinxi.files.wordpress.com/2010/12/clip_image00271.jpg)

Now you can select the tests you want to run and the press Run:

![clip_image002[9]](http://kevinxi.files.wordpress.com/2010/12/clip_image00291.jpg)

After all the tests are run, you can dismiss Nunit. Ncover will produce a file called “Coverage.xml” and exit.

Another free tool, called NcoverExplorer is available to view the Coverage.xml file. To use NcoverExplorer simply choose File – Open and navigate to the Coverage.xml file that you wish to view. The tool will show a tree view of the software in the left-hand pane and a view of the source code in the lower right pane:

![clip_image002[11]](http://kevinxi.files.wordpress.com/2010/12/clip_image002111.jpg)

In the tree view, the color codes have the following meaning:

  • Gray text - the namespace/class/method has zero coverage. (Icon will be grayed also) Red text - the namespace/class/method has partial coverage. Blue text - the namespace/class/method coverage has partial coverage exceeding a satisfaction threshold (default 95%).* Black text - the namespace/class/method has 100% coverage.

Once you have a coverage file open, you may select File – Merge and open another coverage file. By doing this you can see the combined coverage results from two different sets of tests. For example, you can merge the eFinder Unit Test coverage with the Pelican Unit Test coverage to see how Pelican components are covered across all testing.

The satisfaction threshold may be changed in the View – Options window:

![clip_image002[13]](http://kevinxi.files.wordpress.com/2010/12/clip_image00213.jpg)

The source code highlighting works like so:

  • Blue background - this section of code was visited (a tooltip tells you how many times).* Red background - this section of code was not visited.

NcoverExplorer also permits you to create html report of the code coverage. If you select View – Reports and the select Module/Class Summary like so:

![clip_image002[15]](http://kevinxi.files.wordpress.com/2010/12/clip_image00215.jpg)

You will get a CoverageReport.html file that looks like this:

![clip_image002[17]](http://kevinxi.files.wordpress.com/2010/12/clip_image002171.jpg)

Running from the Command Line

Ncover and Nunit may be run from the command line, which is what would probably be done for post-build testing in the Build environment.

Here is an example of how to run Nunit from the command line:

nunit-console.exe /labels PelicanUnitTests.dll > UnitTestResults.txt

Again, if Nunit is not in your path, you will have to provide the complete path. The resulting test results file will look something like this:

![clip_image002[19]](http://kevinxi.files.wordpress.com/2010/12/clip_image002191.jpg)

If you want to combine this with coverage analysis, you could do something like the following. (Note: the command would be all on one line)

Ncover.console.exe //x PelicanCoverage.xml nunit-console.exe /labels PelicanUnitTests.dll > UnitTestResults.txt

The //x option specifies that we want to store the coverage data in the file “PelicanCoverage.xml”.

The file “PelicanCoverage.xml” can then be browsed using NcoverExplorer, however the source files must be in sync with the coverage file in order for this to be effective.

NcoverExplorer may be run from the command line to generate a coverage report. (Command would be all on one line)

NcoverExplorer.console.exe /html:PelicanCoverage.html /report:4 PelicanCoverage.xml

The /html option means that we want to generate a coverage report in HTML format and call it PelicanCoverage.html. The option /report:4 means that we would like to generate a Module and Class summary. The final argument is the Ncover-generated coverage file that we would like to parse.

Design Pattern Series – Singleton

2010-12-27
  • Basic Theory
    • Design Patterns
      • Programming
  • C#
  • Design Pattern Series
  • Singleton

Problem

We need such a service, which keeps only one instance in memory during whole lifetime of the application; we may also need to visit such instance everywhere in the project. A possible solution might be using static global variable, or a new allocated object and pass its point to the place we want. Unfortunately, such design can’t ensure ‘1-instance’ and makes maintenance more difficultly. 

Forces

Any of the following conditions justifies using the solution described in this pattern:

  • You want to keep only one instance of a service or component in application level.
  • You want to make a global service or component easily to be visited  everywhere with less considering of thread/concurrent problems.

    Solution

    Typically, we define private constructors to avoid multiple instance, and declare a private static instance inside of the class, such instance can be visit through a public method.

For .Net/C#, common language runtime help us to keep thread safety for static variables, and here we use a nested class to provide a fully lazy instantiation.

Code Snippet

/// <summary>
/// Thread safety with Generic.
/// </summary>
/// <typeparam name=”T”>Specific Singleton type</typeparam>
public sealed class SingletonT<T>
{
    public static T Instance
    {
        get { return Nested._instance; }
        private set { Nested._instance = value; }
    }
 
    private class Nested
    {
        internal static T _instance =
            (T)Activator.CreateInstance(typeof(T));
    }
}

Liabilities

The Singleton pattern has the following liabilities

  • User should not use keyword ‘new’ on the types with singleton pattern.

  • In native languages, user could release the singleton object outside of its definition, this makes the pointer point to an invalid address.

    Related Patterns

    The following patterns are related to the Singleton pattern:

  • Multiton pattern: http://en.wikipedia.org/wiki/Multiton_pattern

本文仅介绍中缀表达式转换为后缀表达式的原理。

例子： A+B(C-D)-E/F => ABCD-+EF/-

在中缀表达式转换为后缀表达式的过程中，操作数的顺序是一致的。转换的过程首先自左向右读取中缀表达式，遇到操作数就直接输出。遇到操作符时，把操作符压入符号栈，根据优先级比较后决定是否弹栈。

  • isp: in stack priority. 栈内优先级
  • icp: in coming priority. 栈外优先级

    优先级表格：

操作符 # ( ^ * / % + - )
isp 0 1 7 5 3 8
icp 0 8 6 4 2 1
