Josephus Problem

2010-12-25
  • Algorithms
    • Programming
  • Algorithms
  • Josephus

问题描述：n个人（编号0~(n-1))，从0开始报数，报到(m-1)的退出，剩下的人继续从0开始报数。求胜利者的编号。

我们知道第一个人(编号一定是m%n-1) 出列之后，剩下的n-1个人组成了一个新的约瑟夫环（以编号为k=m%n的人开始）：k k+1 … n-2, n-1, 0, 1, 2, … k-2，并且从k开始报0。

现在我们把他们的编号做一下转换：
k –> 0
k+1 –> 1
k+2 –> 2
…
k-2 –> n-2
k-1 –> n-1

变换后就完完全全成为了(n-1)个人报数的子问题，假如我们知道这个子问题的解：例如x是最终的胜利者，那么根据上面这个表把这个x变回去：x=(x+k)%n。

递推公式：令f[i]表示i个人玩游戏报m退出最后胜利者的编号，最后的结果自然是f[n]：

f[1]=0;
f[i]=(f[i-1]+m)%i; (i>1)



#include <stdio.h>




int main()


{


  int n, m, i, s=0;


  scanf(“%d%d”, &n, &m);


  for (i=2; i<=n; i++) s=(s+m)%i;


  printf (“The winner is %d\n”, s+1);


  return 0;


}

Linux下的swap分区是所有分区中唯一一个不需要mount的分区；mount的目的是将分区挂载到目录，由于在Linux里没有盘符的概念，所以通过访问目的达到访问分区内容的目的。

<符号：将某一文件作为重定向后程序的标准输入         如： grep keyword < info.txt
|管道符：将左边的输出作为右边命名的输入                如： ls -a | grep /home

Main函数首先通过失效函数生成fail[]，然后通过fastFind函数返回第一个匹配成功字符串的起始位置。


算法的关键是fail[]的生成，比较难以理解的也是相应失效函数递推计算的部分。






#include <iostream>



#include <string>



using namespace std;



int *fail;//匹配字符串的失效函数



int fastFind(string &cur,string &pat,int *fail){



    int posP=0,posT=0;



    int lengthP=pat.length(),lengthT=cur.length();



    while(posP<lengthP && posT<lengthT){



        if(pat.at(posP)==cur.at(posT)){



            posP++;posT++;//匹配成功一位就同时向后移一位



        }



        else if(posP==0) posT++;//当匹配串没有找到第一个字符时，被匹配字符串向后移



        else posP=fail[posP-1]+1;//否则，通过失效函数找到重新匹配的起始位置



    }



    if(posP<lengthP) return -1;



    else return posT - lengthP;//匹配成功时返回被匹配串中的匹配上的起始位置   



}



/*



失效函数的作用就是计算处fail[]数组，fail[i]的定义为



fail[i] = k (0<=k<i 且 P0P1……Pk = Pi-kPi-k+1……Pi 的最大整数)



        = -1  (其他)



*/



void createFail(string &pat){



    int lengthP = pat.length();



    fail = new int[lengthP];



    fail[0] = -1;



    for(int j=1;j<lengthP;j++){



        int i=fail[j-1];



        while(pat.at(j)!=pat.at(i+1)&&i>=0) i=fail[i];//这一步递推计算失效函数



        if(pat.at(j)==pat.at(i+1)) fail[j]=i+1;



        else fail[j]=-1;



    }



}






int main(){



    string cur,pat;



    getline(cin,cur);getchar();



    getline(cin,pat);



    createFail(pat);



    cout<<fastFind(cur,pat,fail);



    return 0;



}

  About Data Structure


      

      

      

        
  2010-12-25


        
	
	
  • Basic Theory
    • Programming

是数据而不是算法是系统构造的关键因素。  




数据结构是计算机存储、组织数据的方式。通常情况下，精心选择的数据结构可以带来更高的运行或者存储效率的算法。数据结构往往同高效的检索算法和索引技术有关。


Sartaj Sahni 在他的《数据结构、算法与应用》一书中称：“数据结构是数据对象，以及存在于该对象的实例和组成实例的数据元素之间的各种联系。这些联系可以通过定义相关的函数来给出。”他将数据对象（data object）定义为“一个数据对象是实例或值的集合”。
Clifford A.Shaffer 在《数据结构与算法分析》一书中的定义是：“数据结构是 ADT（抽象数据类型 Abstract Data Type） 的物理实现。”
Lobert L.Kruse 在《数据结构与程序设计》一书中，将一个数据结构的设计过程分成抽象层、数据结构层和实现层。其中，抽象层是指抽象数据类型层，它讨论数据的逻辑结构及其运算，数据结构层和实现层讨论一个数据结构的表示和在计算机内的存储细节以及运算的实现。


一般认为，一个数据结构是由数据元素依据某种逻辑联系组织起来的。对数据元素间逻辑关系的描述称为数据的逻辑结构；数据必须在计算机内存储，数据的存储结构是数据结构的实现形式，是其在计算机内的表示；此外讨论一个数据结构必须同时讨论在该类数据上执行的运算才有意义。


在许多类型的程序的设计中，数据结构的选择是一个基本的设计考虑因素。许多大型系统的构造经验表明，系统实现的困难程度和系统构造的质量都严重的依赖于是否选择了最优的数据结构。许多时候，确定了数据结构后，算法就容易得到了。有些时候事情也会反过来，我们根据特定算法来选择数据结构与之适应。不论哪种情况，选择合适的数据结构都是非常重要的。


选择了数据结构，算法也随之确定，是数据而不是算法是系统构造的关键因素。这种洞见导致了许多种软件设计方法和程序设计语言的出现，面向对象的程序设计语言就是其中之一。

This post implements a simple BFS based on queue theory, the following code snippet accept a graph as input, and validate whether it is a connected graph.




#include <iostream>


#include <list>


#include <queue>


using namespace std;




void BFS(int node,list<int> rList,bool isVisited){


    list<int>::iterator i,iend;


    queue<int> q;


    isVisited[node]=true;


    q.push(node);


    while(!q.empty()){


        int a = q.front();


        q.pop();


        iend = rList[a].end();


        for(i=rList[a].begin();i!=iend;i++){


            if(!isVisited[i]){


                isVisited[i]=true;


                q.push(i);


            }


        }


    }


}




int main(){


    int node,edge,j,a,b,ct;


    list<int> rList;


    bool *isVisited;


    while( cin>>node&&node!=0){


        rList = new list<int>[node+1];


        isVisited = new bool[node+1];


        isVisited[0] = true;


        for(j=1;j<=node;j++)


            isVisited[j]=false;




        cin>>edge;


        for(j=0;j<edge;j++){


            cin>>a>>b;


            rList[a].push_back(b);


            rList[b].push_back(a);


        }


        BFS(1,rList,isVisited);


        ct = 0;


        for(j=1;j<=node;j++){


            if(isVisited[j]) ct++;


        }


        if(ct == node) cout<<“Is a Map”<<endl;


        else cout<<“Is not a Map”<<endl;


    }


    return 0;


}

void DFS(int node,list<int> rList,bool isVisited){


    bool haveNode;


    int a;


    int lastNode = new int[node+1];


    for(int j=0;j<=node;j++) lastNode[j]=1;




    list<int>::iterator i,iend;


    stack<int> s;


    s.push(node);


    cout<<“A DFS path is:”<<endl;


    while(!s.empty()){


        a = s.top();


        s.pop();


        isVisited[a]=true;


        cout<<“Search to:”<<a<<endl;


        iend = rList[a].end();


        haveNode = false;


        for(i=rList[a].begin();i!=iend;i++){


            if(!isVisited[i]){


                s.push(i);


                lastNode[i]=a;


                haveNode = true;


            }


        }


        if(!haveNode){


            s.push(lastNode[a]);


            if(lastNode[a]==1){


                bool flag = false;


                iend = rList[1].end();


                for(i=rList[1].begin();i!=iend;i++){


                    if(!isVisited[*i])


                        flag = true;


                }


                if(!flag){


                    cout<<“Back to 1”<<endl;


                    break;


                }


                s.push(1);


            }


        }


    }


    int ct = 0;


    for(int j=1;j<=node;j++){


        if(isVisited[j]) ct++;


    }


    if(ct == node) cout<<“Is a Map”<<endl;


    else cout<<“Is not a Map”<<endl;


}

void setFlag(int node,list<int> rList,bool isVisited,int lastNode){

}

最近一个月，感觉自己像是人间蒸发掉了一样，不上QQ，不发短信，也不打电话，所有的联络都被自己主动的掐断。而在这样封闭的生活里，一个月的时间瞬间即逝。我的改变让家人有点担心了，心里不想让他们知道我的难处，可是也不希望对他们说谎话，就这样主动的减少一些联络，甚至冷淡一些。




让一个人完完全全蒸发是不可能的，人活着就会思考，追求，生活也会有寄托，这样无声无息的一个人生活实在难受，内心从未像现在这样渴望与人说话。五一节的时候回了合肥，顺道回家呆了两天，因为姐姐结婚的缘故，我热热闹闹的折腾了一番，帮前帮后，然后回家陪父母呆了两天，之后就又回来继续一个人的生活。


从四月底到五一前的那一个星期，我基本是倒时差的过的，没有了工作就没有了固定的时间表，困到不行才睡，加上原本要做的外包软件需要赶工，生物钟十分混乱。五月五号回上海后，正常的过了几天，之后又混乱起来，晚上无聊就看看爱情动作片，之后再无聊就看综艺、韩剧，就这样通过狭小的网络打发时间；有时候一天睡四个小时，有时候睡十六个小时，有时候懒的连出去吃饭都不愿意，有时候又吃零食吃到撑；我从未这样放纵过，一个人的世界里，没有约束，没有顾虑，没有快乐，当然也没有朋友分享心情。


前天晚上去老胡那边吃饭，我们几个人自己买菜做饭，一个人一瓶啤酒，我吃好喝完之后躺了一会；可能是睡眠不足，又吃了好多米饭，最后还是吐了。之后11:20左右坐977回来，竟然又没有了困意，躺在床上，一直看网络剧到第二天中午。


几个星期前就开始上网找外包项目，特别是在国外的rentacoder上转悠，我的感受很明显，想透过写外包软件做自由职业者，是很难维持收入的。我通常把自己对创业的想法扼杀掉，特别是在目前这种封闭式的生活下，更加没有可能性。


我想我是在完全没有整理思绪的情况下写下这篇文章的，每次写这种流水账文章都有着几乎相同的原因，那就是心里有某种想法在滋生，或者是原本平静的心情被起了波澜。这一次，我在思考的是，当我手上唯一的外包项目做完我对工作就没有了寄托，当我在这一个月体验完所有的填补空虚的方法后，接下来我该怎么办。那时的生活会不会变得更加的难以忍受？或者我可以很快找到新的目标？

回忆让我拾起起很多零散的片段，在这里拼凑在一起。
最近在校内上看到了一篇文章“中国姓氏英文翻译”，才知道我的姓“奚”翻译成英语通常是Hsi或者Chi，而不是拼音Xi，那这么说我的英文名就应该是Kevin Hsi/Chi了。




“奚”是一个疑问词，可以意译为What/Why/Where/How，而“昊”则是一个名称，可意译为Clear Summer Sky，也有宽广、广大的意思。合起来，我的名字大致就是Wish a clear summer sky，也就是对晴天的期待，就像晴天娃娃(Teru-Teru-Bouzu)一样。


多想用Teru-Teru-Bouzu做我WP博客的名字，可惜不巧有人捷足先登；下面罗列一下我用过的几个网名：


    

  • 关于“haozi”：高中的时候，朋友们给我起的外号：“昊子”，人生第一个外号，第一个不雅的外号。
    • 

  • 关于“cose_haozi”：大二的时候，跟风把学院的缩写加在网名前面(CoSE, College of Software Engineering)。
